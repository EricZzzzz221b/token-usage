# 开发计划

## 1. 节奏原则

- 以可演示的垂直闭环推进，不先堆积大量 UI
- 每个阶段结束都产生可运行构建
- OAuth 安全与脱敏测试从第一阶段开始，不留到发布前补做
- 液态玻璃效果建立在清晰可用的基础布局之后
- 每周只承诺一个主要里程碑，并保留修复和验证时间

## 2. 建议迭代安排

以下按一名全职开发者估算，共约 6 周。多人协作时可以压缩日历时间，但不建议跳过退出条件。

### 第 1 周：基础与数据原型

- 初始化 Tauri、React、TypeScript 和 Rust 工程
- 配置 lint、format、test 和 CI
- 建立内部数据模型和错误模型
- 实现 `auth.json` 解析与 `CODEX_HOME`
- 实现 mock usage API client
- 完成敏感信息日志审计测试

里程碑：使用测试数据完整跑通 Rust 到 UI。

### 第 2 周：真实 OAuth 用量

- 实现 macOS Keychain 读取
- 实现文件回退和凭据状态分类
- 接入官方 usage 端点
- 标准化主、次窗口和未知周期
- 完成 401、403、429、5xx、超时和畸形响应测试
- 加入首次授权说明

里程碑：已登录用户能够看到真实用量。

### 第 3 周：菜单栏 MVP

- 菜单栏模板图标和动态百分比
- 菜单弹层与多进度条
- 手动刷新、自动刷新和并发合并
- 自定义刷新间隔
- last-good、未更新和过期状态
- 中英文基础文案

里程碑：状态栏版本可以日常使用。

### 第 4 周：桌面浮窗

- 无边框透明窗口
- macOS 原生玻璃材质
- 紧凑与详细模式
- 置顶、锁定、鼠标穿透
- 位置和多显示器恢复
- 主题、玻璃强度和透明度
- 减少透明度与减少动态效果适配

里程碑：完成产品核心视觉体验。

### 第 5 周：通知与长期运行

- 阈值状态机和通知去重
- 开机启动
- 睡眠唤醒与网络恢复
- 凭据更新后的自动恢复
- 快照存储和清理
- 诊断导出

里程碑：连续运行和系统事件处理完整。

### 第 6 周：发布准备

- 可访问性检查
- 性能、内存和空闲 CPU 基准
- 安全与隐私检查
- 7 天长稳测试启动并处理发现的问题
- 应用图标与发布素材
- 签名、notarization 和 DMG
- 安装、升级和卸载验证

里程碑：生成 v0.1.0 候选版本。

## 4. Definition of Done

每个工作包完成需要满足：

- 行为与产品规格一致
- 关键逻辑具有自动化测试
- 中英文文案完整
- 错误状态可见且可恢复
- 不向日志写入敏感信息
- 通过 lint、format、unit test 和构建
- 涉及 UI 时完成浅色、深色和至少一种可访问性模式检查
- 文档随行为同步更新

## 5. 风险与缓解

### 内部用量接口变化

缓解：原始响应只在 provider 内解析；使用兼容字段和响应 fixture；UI 只依赖稳定内部模型。

### OAuth token 风险

缓解：只读、短生命周期、固定官方域名、字段白名单诊断、日志测试和首次明确授权。

### macOS 玻璃效果兼容性

缓解：以可读的半透明背景作为回退；分别验证不同 macOS 版本、减少透明度和高对比度。

### 菜单栏动态文字宽度

缓解：使用固定格式和等宽数字；提供仅图标模式；100% 后使用明确的紧凑表现。

### 长期运行状态漂移

缓解：集中式刷新调度、单飞请求、睡眠恢复、网络恢复和 7 天长稳测试。

## 6. 首个开发切片

第一个可执行切片应同时覆盖：

1. 读取测试凭据 fixture。
2. 请求本地 mock usage endpoint。
3. 标准化两个额度窗口。
4. 在开发页面显示两条进度条。
5. 验证日志中不存在 token。

完成这个切片后，再接触真实 Keychain 和官方端点，能显著降低凭据处理和 UI 同时调试的风险。
