# ADR 0002: 首版使用直接 OAuth 用量数据源

- 状态：已接受
- 日期：2026-07-11

## 决策

MVP 只实现从本机 Codex OAuth 登录态读取凭据并查询官方用量端点。架构保留 `UsageProvider` 接口，但暂不实现 app-server 数据源。

## 原因

- CC Switch 已验证该路径可用于官方订阅额度展示
- 实现直接、依赖少，适合尽快形成真实用量闭环
- 产品只读监控，不承担登录、代理或供应商切换职责

## 后果

- 必须严格保护 OAuth token，不得持久化或记录
- 上游内部响应变化需要通过 provider 适配层吸收
- 后续可加入 app-server provider，而不改变 UI 数据模型
