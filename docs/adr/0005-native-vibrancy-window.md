# ADR 0005: 原生 vibrancy 浮窗

- 状态：已接受
- 日期：2026-07-11

## 决策

macOS 浮窗使用 `window-vibrancy` 在 Tauri 窗口上安装原生 `NSVisualEffectView`，前端 CSS 负责玻璃高光、描边和内容层级。窗口位置与尺寸由官方 `tauri-plugin-window-state` 保存。

## 原因

- 原生 vibrancy 能根据桌面背景和系统外观产生真实材质效果
- 仅用 CSS `backdrop-filter` 无法稳定读取 Tauri 窗口背后的桌面内容
- 原生材质与 CSS 叠加可以形成接近液态玻璃的层次，同时保留可读性回退
- 官方窗口状态插件可以正确处理多显示器物理位置和尺寸

## 后果

- macOS 是首发和主要验证平台
- 清透、标准、浓郁分别映射到不同语义材质
- 减少透明度时前端必须提供不依赖背景的实色回退
- 鼠标穿透后必须可通过菜单栏恢复交互
- 紧凑模式必须可通过菜单栏切回详细模式
