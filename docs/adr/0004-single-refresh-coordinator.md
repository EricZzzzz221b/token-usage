# ADR 0004: 单一刷新协调器

- 状态：已接受
- 日期：2026-07-11

## 决策

所有官方用量请求由 Rust `RefreshCoordinator` 统一调度。菜单栏、浮窗和设置界面只能读取协调器快照或请求协调器刷新，不得分别调用官方接口。

## 原因

- 避免菜单栏和浮窗同时刷新造成重复官方请求
- 为自动刷新、手动刷新合并和 last-good 策略提供唯一状态源
- 便于后续加入睡眠恢复、网络恢复和通知阈值判断
- 前端重新加载不会丢失后台快照

## 后果

- 后台任务在应用启动时立即刷新，此后按设置间隔运行
- 状态变化通过 `usage://updated` 广播给所有前端窗口
- 瞬时错误保留最近成功快照，确定性凭据错误立即清除旧快照
- 刷新间隔保存在应用配置目录，允许范围为 1–1440 分钟
