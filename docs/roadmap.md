# 开发路线

## 阶段 0：项目基础

目标：建立可重复开发、测试和发布的项目骨架。

交付：

- Tauri 2 + React + TypeScript 工程
- Rust、TypeScript lint 与格式化
- 基础 CI
- 中英文 i18n
- 应用标识、版本和配置约定
- ADR（架构决策记录）目录

退出条件：全新环境可以完成安装、开发启动、测试和打包。

## 阶段 1：真实用量闭环

目标：从本机 OAuth 登录态得到真实用量数据。

交付：

- Keychain 和 `auth.json` 只读凭据发现
- `CODEX_HOME` 支持
- 官方用量请求
- 响应标准化
- 错误分类
- Mock 测试
- 最小开发调试页面

退出条件：已登录用户能稳定看到真实额度窗口；日志中没有 token。

## 阶段 2：菜单栏 MVP

目标：形成最小可日常使用产品。

交付：

- 菜单栏图标和最高用量百分比
- 菜单弹层中的多窗口进度条
- 重置倒计时
- 手动刷新
- 5 分钟默认自动刷新
- 刷新间隔设置
- last-good 与数据过期状态

退出条件：不打开主窗口即可完成核心查看和刷新操作。

## 阶段 3：液态玻璃浮窗

目标：实现产品的核心视觉与桌面常驻体验。

交付：

- 原生玻璃材质
- 紧凑与详细模式
- 普通置顶
- 拖动、锁定与鼠标穿透
- 透明度和玻璃强度
- 位置、尺寸和显示器恢复
- 浅色、深色、高对比度与减少透明度适配

退出条件：浮窗在常见桌面背景、多显示器和睡眠恢复场景下稳定可读。

## 阶段 4：通知与系统集成

目标：完善长期常驻能力。

交付：

- 90%、100% 阈值通知
- 可选 70% 和重置通知
- 通知去重
- 开机启动
- 唤醒和网络恢复刷新
- 隐藏 Dock 图标设置
- 首次启动授权说明

退出条件：连续运行与日常系统事件不会产生重复通知或失效状态。

## 阶段 5：质量与发布

目标：形成可分发的首个版本。

交付：

- VoiceOver 和键盘可访问性
- 诊断导出
- 隐私说明
- 资源占用基准
- 7 天长稳测试
- Apple 签名和 notarization
- DMG 发布物
- 安装、升级和卸载验证

退出条件：所有 P0/P1 缺陷关闭，安全清单通过，候选版本完成签名和公证。

## 后续版本候选

- 本地历史曲线
- app-server 数据源
- Credits 信息
- Windows 版本
- 自动更新
- 多账号只读监控
