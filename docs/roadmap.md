# 开发路线

## 阶段 0：项目基础

状态：已完成（2026-07-11）。

目标：建立可重复开发、测试和发布的项目骨架。

交付：

- Tauri 2 + React + TypeScript 工程
- Rust、TypeScript lint 与格式化
- 基础 CI
- 中英文 i18n
- 应用标识、版本和配置约定
- ADR（架构决策记录）目录

退出条件：全新环境可以完成安装、开发启动、测试和打包。

完成证据：

- Node 22 与 Rust stable 工具链约束
- npm 与 Cargo 锁文件
- Prettier、ESLint、TypeScript、Vitest、rustfmt 与 Clippy
- macOS GitHub Actions CI
- 中英文基础资源和可访问进度条示例
- Tauri release 构建成功
- ADR 0001 与 ADR 0002

## 阶段 1：真实用量闭环

状态：已完成（2026-07-11）。

目标：从本机 OAuth 登录态得到真实用量数据。

交付：

- Keychain 和 `auth.json` 只读凭据发现
- `CODEX_HOME` 支持
- 官方用量请求
- 响应标准化
- 错误分类
- Mock 测试
- 最小开发调试页面

退出条件：已登录用户能稳定看到真实额度窗口；日志中没有 token。

完成证据：

- macOS Keychain 与 `CODEX_HOME/auth.json` 只读凭据发现
- 仅接受 `auth_mode = chatgpt` 的官方 OAuth 登录态
- 固定 `chatgpt.com` 用量端点且禁止 HTTP 重定向
- Authorization header 标记为敏感值，不进入日志或错误载荷
- 5 小时、7 天、30 天与动态周期标准化
- 401/403、429、5xx、网络和响应不兼容错误分类
- Rust 本地 mock 验证请求头、成功响应与认证失败
- 前端加载、成功与可恢复错误状态
- 本机官方订阅真实端点只读验证成功，未输出凭据或账号信息

## 阶段 2：菜单栏 MVP

状态：已完成（2026-07-11）。

目标：形成最小可日常使用产品。

交付：

- 菜单栏图标和最高用量百分比
- 菜单弹层中的多窗口进度条
- 重置倒计时
- 手动刷新
- 5 分钟默认自动刷新
- 刷新间隔设置
- last-good 与数据过期状态

退出条件：不打开主窗口即可完成核心查看和刷新操作。

完成证据：

- macOS 菜单栏图标与动态最高使用率标题
- 托盘 tooltip 和菜单摘要展示所有额度窗口
- 托盘菜单支持显示窗口、立即刷新和退出
- Rust 单一刷新协调器，托盘与前端共享同一份快照
- 默认 5 分钟自动刷新，支持 1、5、10、15、30、60 分钟设置
- 设置持久化到应用配置目录，限制为 1–1440 分钟
- 同时发起刷新时复用当前状态，避免并发重复请求
- 瞬时错误保留 last-good，认证和凭据错误立即透出
- 前端订阅 `usage://updated` 事件并支持手动刷新
- 10 个 Rust 测试与 3 个前端交互测试

## 阶段 3：液态玻璃浮窗

状态：已完成（2026-07-11）。

目标：实现产品的核心视觉与桌面常驻体验。

交付：

- 原生玻璃材质
- 紧凑与详细模式
- 普通置顶
- 拖动、锁定与鼠标穿透
- 透明度和玻璃强度
- 位置、尺寸和显示器恢复
- 浅色、深色、高对比度与减少透明度适配

退出条件：浮窗在常见桌面背景、多显示器和睡眠恢复场景下稳定可读。

完成证据：

- macOS 原生 `NSVisualEffectView` vibrancy 材质
- 清透、标准、浓郁三档玻璃强度
- 紧凑与详细两种浮窗尺寸和信息层级
- 顶部拖动区域、锁定位置和总在最前
- 鼠标穿透及菜单栏“恢复浮窗交互”安全出口
- 菜单栏支持切换紧凑/详细模式，避免紧凑模式失去设置入口
- 55%–100% 透明度设置
- 官方窗口状态插件保存并恢复位置与尺寸
- 进度条按正常、注意、紧张状态切换颜色
- 详细模式展示剩余百分比和重置时间
- 浅色、深色、减少透明度和减少动态效果适配
- 12 个 Rust 测试、3 个前端浮窗交互测试和真实 macOS 启动验证

## 阶段 4：通知与系统集成

状态：已完成（2026-07-11）。

目标：完善长期常驻能力。

交付：

- 90%、100% 阈值通知
- 可选 70% 和重置通知
- 通知去重
- 开机启动
- 唤醒和网络恢复刷新
- 隐藏 Dock 图标设置
- 首次启动授权说明

退出条件：连续运行与日常系统事件不会产生重复通知或失效状态。

完成证据：

- 70%、90%、100% 与重置通知开关
- 同一额度周期和阈值只通知一次，周期变化后自动清理状态
- 系统通知权限显式请求
- 登录时启动设置
- 睡眠恢复后由逾期调度器立即刷新，瞬时网络错误 30 秒探测恢复
- 每次请求重新读取凭据，Codex 登录更新后无需重启应用
- 30 天非敏感 JSONL 快照历史与 8,640 条上限
- 首次启用前展示凭据用途和固定官方域名说明
- 关闭主窗口时隐藏，菜单栏继续常驻
