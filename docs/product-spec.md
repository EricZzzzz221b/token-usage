# Token用量产品规格

## 1. 产品定义

Token用量（英文名 Token Usage）是一款面向 Codex 用户的跨平台状态栏工具。用户无需反复打开 Codex，即可查看当前用量、重置时间、账号模式和本地任务状态。

产品需要帮助用户快速回答四个问题：

1. 当前已经使用多少额度？
2. 各额度窗口何时重置？
3. 当前是否需要控制使用节奏？
4. 我离开 Codex 后，任务进行到哪一步、是否已经完成？

## 2. 目标用户

- 已通过 Codex CLI 或 Codex App 登录 ChatGPT 官方账号
- 使用支持 Codex 的官方 ChatGPT 订阅
- 长时间或高频使用 Codex
- 希望在 macOS 桌面持续观察用量变化
- 会同时运行一个或多个 Codex 任务，希望切到其他工作后仍能掌握进度

## 3. MVP 范围

### 包含

- macOS Apple Silicon 首发版本
- Codex 官方 OAuth 凭据发现
- 官方订阅用量查询
- 5 小时、7 天、30 天及未知周期的动态展示
- 菜单栏状态与快捷菜单
- 可选桌面浮窗
- 紧凑与详细两种浮窗模式
- 多周期进度条
- 已用百分比、剩余百分比和重置倒计时
- 手动刷新和可配置自动刷新
- 最后成功数据保留与过期标记
- 90% 和 100% 默认通知
- 开机启动
- 简体中文和英文
- 浅色、深色和跟随系统主题
- 诊断信息导出
- 自动识别订阅/API 模式
- API 模式 Credits 展示
- 官方使用限额重置机会
- 多任务实时状态、运行时长和完成通知
- 最近 5 条完成记录及会话跳转

### 不包含

- Windows ARM64 版本
- Credits 购买或充值
- 第三方供应商或中转服务
- 多账号管理和账号切换
- OAuth 登录流程
- 请求代理或网络路由
- Credits 购买
- 云同步
- 历史图表
- 自动更新（在签名与发布流程稳定后加入）

## 4. 核心体验

### 4.1 首次启动

首次启动必须先显示本地凭据使用说明，用户主动点击“启用用量查询”后才能读取凭据并请求用量。说明需要明确：

- 读取本机已有 Codex 登录凭据
- 仅向 `chatgpt.com` 查询订阅用量
- 不保存、上传或分享凭据

未发现登录态时，引导用户通过 Codex CLI 或 Codex App 完成官方登录，不在本产品内实现 OAuth。

### 4.2 菜单栏

菜单栏是产品主体，默认随应用启动。提供三种显示方式：

- 仅图标
- 图标与最高使用率
- 仅最高使用率

用户可选择状态栏显示 5 小时或 7 天窗口。存在活动任务时，状态栏同时显示任务状态和运行时间；多个任务分别保留在任务面板中。点击菜单栏后显示用量摘要、浮窗开关、立即刷新、设置和退出。

### 4.3 桌面浮窗

浮窗默认关闭，用户可主动开启。默认行为：

- 普通置顶
- 全屏应用可覆盖
- 记住位置、尺寸和显示器
- 支持锁定位置和鼠标穿透
- 支持紧凑与详细模式
- 使用稳定、清晰的标准外观

紧凑模式展示当前任务状态、运行时间和所选额度窗口。详细模式提供“用量/任务”标签：用量页展示剩余百分比、重置日期、重置机会和最后更新时间；任务页分别展示进行中任务和最近 5 条完成记录。

## 5. 视觉标准

视觉针对桌面常驻工具保持安静、清晰和低干扰。

- 使用稳定的标准面板和明确的信息层级，不提供通透强度调节
- 详细模式以额度与任务内容为主，账号模式作为顶部辅助信息
- 紧凑模式保持单行结构，避免堆叠次级信息
- 不使用大面积彩色渐变、持续流光或循环警示动画
- 所有文字必须在不同桌面背景下保持可读
- 减少动态效果系统设置开启时，禁用非必要动画

进度条标准：

| 使用率       | 状态     | 色彩语义 |
| ------------ | -------- | -------- |
| 0–69%        | 充足     | 薄荷绿   |
| 70–89%       | 注意     | 琥珀黄   |
| 90–99%       | 紧张     | 珊瑚红   |
| ≥100%        | 已达限制 | 深红     |
| 无数据或过期 | 未知     | 银灰     |

状态不能仅依赖颜色，必须同时提供文字、数字或图标。

## 6. 刷新规则

- 默认自动刷新间隔为 5 分钟
- 用户可选择关闭、1、5、10、15、30、60 分钟或输入自定义值
- 自定义范围为 1–1440 分钟
- 应用启动、设备唤醒和网络恢复后触发一次刷新
- 同一时刻只能存在一个查询请求
- 多次手动刷新需要合并
- 查询超时默认 15 秒
- 瞬时失败自动重试一次

任务状态独立于用量接口，每 2 秒检查本机会话变化。扫描器共享内存快照并按字节增量读取 JSONL；标题变化不能触发完整日志重读。

数据陈旧规则：

- 10 分钟内：继续正常展示，并显示最后更新时间
- 10–30 分钟：保留数据，增加“未更新”提示
- 超过 30 分钟：进度条变灰，显示“数据已过期”
- 401、403、凭据缺失或解析失败：立即显示确定性错误

## 7. 通知规则

- 70% 通知默认关闭
- 90% 通知默认开启
- 100% 通知默认开启
- 重置通知可选
- 每个额度窗口独立计算阈值
- 同一周期内同一阈值只通知一次
- 检测到周期重置后清除该窗口的通知状态

## 8. 可访问性

- 支持 VoiceOver 标签和键盘操作
- 支持增加对比度与减少透明度设置
- 支持减少动态效果
- 百分比使用等宽数字，避免刷新时布局跳动
- 菜单栏和浮窗必须在浅色、深色与高对比度模式下可辨识

## 9. 成功标准

- 用户首次启动后 60 秒内成功看到真实订阅用量
- 常驻空闲时内存和 CPU 占用保持轻量
- 自动刷新不打断用户操作，不出现窗口闪烁
- 网络中断不会清空最后可信数据
- 所有日志与诊断文件均不包含 OAuth token
- 连续运行 7 天后浮窗位置、刷新和睡眠恢复保持正常
